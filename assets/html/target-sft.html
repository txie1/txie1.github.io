Overview

TL;DR

SFT losses are usually studied as objectives, but their real effect is to define a token-level target distribution through gradients. We reinterpret SFT as target distribution design, introduce the Q-target framework that unifies existing methods, and propose Target-SFT — which adaptively combines label imitation with teacher-guided alternatives.
  • 1 Standard SFT is implicitly one-hot target matching — every observed token is treated as the uniquely correct continuation.
  • 2 Many existing SFT variants can be unified as different implicit choices of a target distribution $Q = \gamma_t \delta_{y_t} + (1-\gamma_t)\tilde\pi_t$.
  • 3 Target-SFT explicitly designs both branches: label trust $\gamma_t$ via model probability, and the residual $\tilde\pi_t$ via a teacher-guided distribution.
  • 4 Target-SFT achieves the best Average@16 across all 10 evaluated dataset-model settings in math and medical reasoning.
Motivation

A loss is a surrogate. What target does it imply?

Every SFT loss generates gradients over the vocabulary. Those gradients reveal the target distribution the model is being pushed toward.

At each prefix $x_t$, the model outputs logits $z$ and a distribution $p = \text{softmax}(z)$. The cross-entropy gradient with respect to logit $k$ is:

$$\frac{\partial \mathcal{L}_{\text{CE}}}{\partial z_k} = p_k - Q_k$$

This relationship can be inverted. Given any differentiable loss $\mathcal{L}$ with logit gradient $g_k = \partial \mathcal{L}/\partial z_k$, the induced target distribution is:

$$Q_k := p_k - g_k$$

This reveals what each loss is actually teaching the model — not via its algebraic form, but via the probability updates it drives.

The loss is the mechanism. The induced target distribution is the learning signal.
Example

Standard SFT induces a one-hot target

Applying the inversion formula to standard cross-entropy reveals its hidden assumption.

Standard SFT minimizes $\mathcal{L}_\text{SFT} = -\log \pi_\theta(y_t \mid x_t)$, whose logit gradient is $g_j = p_j - \mathbf{1}\{j = y_t\}$. Applying $Q_k = p_k - g_k$:

$$Q_\text{SFT}(k) = \begin{cases} 1 & k = y_t \\ 0 & k \neq y_t \end{cases} = \delta_{y_t}$$

The induced target is exactly the one-hot distribution — assigning all probability mass to the demonstrated token and none to any alternative.

Target distribution induced by standard SFT
All target mass placed on the observed token $y_t$. Every other token receives $Q = 0$.
Implicit assumption: Every demonstrated token is ideal, reliable, and the uniquely correct continuation — regardless of noise, ambiguity, or the model's existing knowledge.

Contrast this with p-loss — a probability-weighted loss $\mathcal{L} = -\mathrm{sg}(p_y)\log\pi_\theta(y_t)$. Its logit gradients are $g_{y_t} = -p_y(1-p_y)$ and $g_k = p_y\,p_k$. Applying $Q_k = p_k - g_k$:

$$Q_\text{p-loss}(k) = \begin{cases} 2p_y - p_y^2 & k = y_t \\ (1-p_y)\,p_k & k \neq y_t \end{cases}$$

Unlike SFT, p-loss contracts the correct-token target and spreads mass to alternatives when the model is uncertain. The chart below uses a binary vocabulary to make this concrete.

Framework

Relax one-hot supervision by modeling label uncertainty

We introduce a two-parameter target distribution that separates label trust from residual probability allocation.

We replace the rigid one-hot target with a mixture distribution $Q_t$ controlled by two explicit design choices:

$$Q_t = \gamma_t \,\delta_{y_t} + (1 - \gamma_t)\,\tilde{\pi}_t$$
Component Question it answers Effect
$\gamma_t \in [0,1]$ How much should we trust the observed token $y_t$? Controls imitation strength
$\tilde{\pi}_t \in \Delta^{|\mathcal{V}|}$ Where should the remaining $(1-\gamma_t)$ mass go? Shapes alternative supervision

The training objective under $Q_t$ decomposes cleanly:

$$\mathcal{L}_Q(\theta) = \gamma_t \underbrace{\text{CE}(\delta_{y_t},\, \pi_\theta)}_{\text{imitate } y_t} + (1-\gamma_t) \underbrace{\text{CE}(\tilde\pi_t,\, \pi_\theta)}_{\text{match alternatives}}$$

This perspective shows that SFT training is always balancing two forces — imitation of the label and matching of a residual distribution. Standard SFT simply sets $\gamma_t = 1$, collapsing the second term entirely.

Unifying View

Existing SFT variants are implicit Q-target designs

Viewed through the Q-target lens, seemingly different methods differ only in how they specify $\gamma_t$ and $\tilde\pi_t$.

Method

Design both branches of the target

Target-SFT instantiates the Q-target framework by making principled choices for both $\gamma_t$ and $\tilde\pi_t$.

Target-SFT Adaptive fallback supervision
$$Q_t^\text{TARGET} = p_y\,\delta_{y_t} + (1-p_y)\,\tilde{\pi}_t^\text{guided}$$
Label Trust
$$\gamma_t = p_y = \pi_\theta(y_t \mid x_t)$$
Model's own probability on $y_t$ as a proxy for label reliability. High confidence → high trust.
Residual Distribution
$$\tilde{\pi}_t^\text{guided}(a) \propto \pi_\theta(a)^{1-\eta}\,\pi_T(a)^\eta$$
Geometric interpolation between student and teacher. $\eta \in [0,1]$ controls teacher intensity.

Adaptive behavior

High $p_y$ (model is confident) Objective approaches standard SFT. The model already assigns high probability to $y_t$, so we trust the demonstrated token and imitate it closely.
Low $p_y$ (model is uncertain) Imitation weakens; teacher-guided residual gets higher weight. The teacher provides corrective fallback supervision toward better alternatives.

Unlike full distillation — which uses the teacher as a fixed, uniform replacement for the label — Target-SFT uses the teacher selectively, scaling its influence by $1-p_y$. This avoids over-relying on the teacher when the demonstrated token is already well-supported.

Summary

What should readers take away?

01
SFT is fundamentally a target distribution design problem, not just a choice of loss function.
02
Standard SFT is the one-hot special case: it implicitly assumes every token is the uniquely ideal target.
03
Existing variants differ in how they specify label trust $\gamma_t$ or the residual distribution $\tilde\pi_t$ — often implicitly.
04
Target-SFT designs both branches explicitly and achieves consistent improvements across diverse settings.
Instead of asking only which loss should we use?, we ask: what target distribution should the model learn from each token?
